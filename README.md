# Oasis 安定運用予測ツール（ローカルアプリ）

Discordゲーム「おあしすっち」の3連単を、学習済みモデル＋モンテカルロで予測し、
**安定運用（成立のみ・分数ケリー・資金管理）**で買い目と口数を提案するローカルアプリです。
ベットログは手元のCSVに残るので、予測と実測を答え合わせできます。

---

## 1. 必要なもの

- Python 3.9 以上（推奨 3.10〜3.12）
- レースログのテキスト（Discordエクスポート）: 既定ファイル名 `OASISUTTI.txt`

## 2. セットアップ（初回のみ）

ターミナル（WindowsはコマンドプロンプトかPowerShell）でこのフォルダに移動して:

```bash
pip install -r requirements.txt
```

> 仮想環境を使う場合（任意・推奨）:
> ```bash
> python -m venv .venv
> # Windows:  .venv\Scripts\activate
> # Mac/Linux: source .venv/bin/activate
> pip install -r requirements.txt
> ```

## 3. 起動

```bash
streamlit run oasis_app.py
```

ブラウザが自動で開きます（開かなければ `http://localhost:8501`）。
付属の起動スクリプトでも開けます:
- Windows: `run.bat` をダブルクリック
- Mac: `run.command` をダブルクリック（初回は「実行権限」が必要な場合あり → ターミナルで `chmod +x run.command`）

---

## 4. 使い方

1. **サイドバー「1) モデル学習」** に `OASISUTTI.txt` のパスを入れて **「モデル学習 / 再学習」** を押す（数秒）。
   - `OASISUTTI.txt` をこのフォルダに置けば、パスはファイル名だけでOK。
2. **サイドバー「2) 安定運用パラメータ」** で資金やリスクを設定。
   - **手元資金 BANKROLL**: ケリーサイジングの基準。増減したら更新。
   - **分数ケリー**: 0.25=クォーターケリー（小さいほど低分散）。
   - **1レース上限（資金比）**: そのレースで賭ける総額の上限。
   - **エッジ下限**: これ未満の薄い買い目は見送り。
3. **メインのテキスト欄** にステータス画面＋3連単オッズ（統合フォーマット）を貼り付け、**「解析」**。
4. **推奨配分**（✅）と**口数**、**実効EV / いずれか的中 / 最大損失**が出ます。購入リストはコピー用。
5. レース後、**「実績ログ」** で結果（実着順）を入力して精算 → **「成績レポート」** で
   予測 vs 実測（キャリブレーション）と ROI を確認。

### 未成立スリーブ（任意）

サイドバーの「未成立スリーブ」をONにすると、**市場が張っていない（未成立）組**にも各1口だけ
賭けます。当たれば全プール総取りで実効オッズが非常に高く（例: プール120万で121倍）、
モデルの的中率が一定以上あれば高い+EVになります。

- 採用条件: モデル的中率 ≥ 下限（既定5%）かつ 実効エッジ ≥ 30%（小プールでの過剰投資を防止）。
- 合計口数を **3 / 4 / 5口** で上限設定（各組1口）。最悪損失はその口数×1万に限定。
- **性質はハイリスク**: ほとんど外れ、稀に大勝ち（+EVの宝くじ）。安定収益とは別物です。
  **モデルの的中率が正しい前提**なので、成績レポートで低確率帯（5%帯など）の予測≈実測を
  確認してから使うのが安全です。

### 貼り付けフォーマット（統合）

```
pool=1200000          ← 任意（払戻プール。あると口数計算が正確）
=== 出走馬一覧 ===
レース距離,馬場,馬名,SPEED,POWER,STAMINA,コンディション,パッシブスキル,単勝オッズ
中距離,芝,たかな,48,134,59,好調,🏃‍♂️ 中距離得意,2.14
... （全馬）
=== 3連単オッズ ===
順位,1着,2着,3着,オッズ
1,あかり,たかな,ゲロ,2.95
... （成立分）
```

> 3連単オッズを別CSVで持っている場合は、サイドバー「詳細」にCSVパスを入れてください。

## 5. 安定運用の考え方（重要）

- **未成立（全プール総取り）の高倍率な賭けは対象外**にしています（高分散・モデル依存が強いため）。
- 各買い目は**分数ケリー**で資金に比例して小さく、複数に**分散**して張ります。
- 1レースの総リスクは**資金の上限%以内**に制限します（破産回避）。
- **出力はモデルの的中率が正しい前提**です。まずは小さく回し、**成績レポートで「予測≈実測」**を
  数十レース確認してから資金を増やすのが安全です。校正ヒストに従って σ（SIGMA_OVERRIDE）を
  調整し、再学習してください。

### 予測モデルについて（攻略本の式を反映）

スコア予測は、ログから学習する RandomForest に加えて、**非公式攻略本（氷空氏・6/7時点）由来の
内部式を特徴量として明示的に組み込んで**います。

- 理論スコア = `SPEED×距離係数×パッシブ係数 + POWER×距離係数×パッシブ係数`
  （距離係数: 第3巻、パッシブ係数: 第2巻）
- スタミナ足切り = 距離別の必要最低スタミナ（第1巻: 短30/マイル41/中66/長81）。
  実効スタミナが下回るとスコアが大きく落ちるため、特徴量とUI警告に反映。
- 実ログ5681行で検証: 足切りクリア勢で理論スコアと実スコアの相関 0.96、
  未来レース予測の汎化が改善（時系列R² 0.945→0.958）。
- 攻略本は非公式・6/7時点・近似のため、式は「強い手がかり」として使い、最終的には
  データ学習側が補正します。ゲーム更新で式が変わったら、ログを増やして再学習すれば追従します。

## 6. exe化（Python不要の実行ファイルを作る）

Streamlitはサーバ型のため、PyInstaller でランチャ(`run_app.py`)ごと固める方式です。
**重要: exeはビルドしたOS専用**（Windowsで作ればWindows用、Macで作ればMac用。相互変換不可）。

### 手順

1. このフォルダで:
   - Windows: `build_exe.bat` をダブルクリック（または実行）
   - Mac/Linux: `bash build_exe.sh`
2. 数分のビルド後、`dist/OasisYosou/` フォルダが出来ます。
   - Windows: `dist\OasisYosou\OasisYosou.exe`
   - Mac/Linux: `dist/OasisYosou/OasisYosou`
3. **`OASISUTTI.txt` を `dist/OasisYosou/` に置いて** exe を実行。ブラウザが自動で開きます。
   黒いコンソール窓はローカルサーバ本体です（**閉じるとアプリが止まります**）。
   ベットログ `oasis_bet_log.csv` もこのフォルダに作られます。

### 注意・トラブル対処

- **配布サイズは大きめ**（300〜600MB程度。streamlit/scikit-learn/pyarrow を含むため）。
- 初回起動はやや遅い（展開・読み込みのため）。
- ウイルス対策ソフトが PyInstaller 製exeを誤検知することがあります（除外設定で回避）。
- ビルドで `ModuleNotFoundError` が出たら、`oasis.spec` の `hiddenimports` にそのモジュール名を
  追記して再ビルド。`PyInstaller>=6` を推奨（`python -m pip install -U pyinstaller`）。
- どうしてもexe化が通らない場合は、**`streamlit run oasis_app.py`（Python必要）が確実な代替**です。
- 1ファイル(.exe単体)にしたい場合は `oasis.spec` を onefile 構成に変更できます（起動は遅くなります）。
  必要なら onefile 版の spec も用意します。

## 7. ファイル構成

| ファイル | 役割 |
|---|---|
| `oasis_app.py` | 画面（Streamlit）。`streamlit run` で起動。 |
| `oasis_core.py` | ロジック本体（学習・解析・配分・スリーブ・ベットログ）。UI非依存。 |
| `requirements.txt` | 依存ライブラリ。 |
| `run_app.py` / `oasis.spec` | exe化用のランチャとPyInstaller設定。 |

| `oasis_bet_log.csv` | ベットログ（自動生成）。 |

## 8. よくある質問

- **Q. ログが「見つかりません」になる / ベットログCSVが見当たらない** →
  相対パス（ファイル名だけ）は**アプリのフォルダ**が基準です。`OASISUTTI.txt` をアプリと同じ
  フォルダに置いてください。サイドバーに「アプリの場所」と、ログの**解決後の絶対パス**（✅/❌付き）、
  記録・精算時に**保存先の絶対パス**を表示するので、そこを確認してください。フルパス指定も可能です。
- **Q. 結果を入力して精算してもCSVに反映されない** → 精算は記録したのと同じファイルに保存されます
  （上記の絶対パスを確認）。着順の選択肢は「そのレースで記録した買い目」から自動復元されるので、
  別レースを解析した後でも正しい馬で精算できます。精算後に「的中○点」と保存先が表示されれば成功です。
- **Q. 数字が毎回同じ** → 再現性のためシミュレーションのシードを固定しています（校正のため）。
- **Q. 「未学習パッシブ」警告** → 学習データに無いスキルは勝率に反映されません。ログを増やすか表記を統一。
- **Q. 未成立スリーブで毎回損する** → 性質上ほとんど外れます（稀に大勝ち）。+EVでも分散が大きいので、
  口数上限を3に下げる／OFFにする、または校正を確認してから使ってください。
